# Resumo do Projeto

Sistema de Gestão da Infraestrutura Pública (SGIP)

O SGIP é um sistema web que permite acompanhar, registrar e gerenciar informações relacionadas à infraestrutura pública (ruas, praças, iluminação, transporte urbano etc.). O objetivo é dar mais eficiência à gestão pública, possibilitando que cidadãos e órgãos responsáveis tenham acesso às informações em tempo real.


# Atividades da Etapa 2 (N708)

Planejamento Inicial:
A equipe iniciou a Etapa 2 com uma revisão detalhada dos objetivos gerais do projeto, assegurando alinhamento entre todos os integrantes. Foram definidos os papéis de cada membro, estabelecendo responsabilidades claras para backend, frontend e banco de dados. Também foi organizado um cronograma atualizado, distribuindo as entregas em etapas semanais para garantir eficiência e rastreabilidade do progresso.

Levantamento e Preparação: 

Nesta fase, foi criado o novo banco de dados estruturado de acordo com os requisitos revisados. Os requisitos do sistema foram atualizados após análise das necessidades identificadas na Etapa 1. Casos de uso e fluxo de telas foram revisados e ajustados para garantir coerência com a implementação planejada. A preparação incluiu também a definição das rotas iniciais da API e o planejamento das telas prioritárias do frontend.

Execução:

Durante a execução, as principais funcionalidades foram implementadas conforme planejado:
Backend: Foram desenvolvidas e testadas as APIs responsáveis pelas operações de criação, consulta e atualização (POST/GET/PUT) tanto de ocorrências quanto de usuários. A estrutura do projeto backend foi organizada visando escalabilidade e manutenção.


Banco de Dados: As tabelas definidas no modelo lógico foram criadas e devidamente utilizadas com dados de teste para possibilitar a validação das funcionalidades. As relações entre tabelas foram implementadas conforme o DER revisado.


Frontend: Um protótipo funcional das telas principais foi construído, incluindo interface para cadastro de usuários, registro de ocorrências e visualização das informações. As telas seguem o layout definido nos mockups da etapa de planejamento.



# Validação e Ajustes
